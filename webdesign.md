1\. Typography



&#x20;   Font Selection: Avoid overused defaults. Source fonts from fontshare.com or uncut.wtf.



&#x20;   Type Scale: Establish rules for font sizes, letter spacing, and line height to ensure cohesive text.



&#x20;       Size: Define a base paragraph size (e.g., 16px). Scale headings using a ratio like the Major Third scale (1.25x increase per level). Use rem values instead of pixels for automated calculation.



&#x20;       Letter Spacing: Maintain default spacing for body text to ensure legibility. Reduce spacing slightly as heading sizes increase.



&#x20;       Line Height: Set body text line height to 150% (1.5x) of the font size. Decrease line height for headings to improve scannability.



&#x20;   Automation Tool: Generate type systems instantly at typescale.net.



2\. Layout



&#x20;   Grid System: Structure designs using a 12-column layout for desktop, 8-column for tablets, and 4-column for mobile.



&#x20;   Spacing: Implement an 8-point grid system. Ensure all margins, padding, and spacing are multiples of 8 (8, 16, 24, 32).



&#x20;   Visual Hierarchy: Control user attention through:



&#x20;       Proximity: Group related elements.



&#x20;       Size \& Contrast: Emphasize importance using scale, font weight, color, or opacity reductions.



&#x20;       Alignment: Maintain clean lines for clear structure.



3\. Color



&#x20;   Limits: Restrict palettes to 2-3 functional colors.



&#x20;   60-30-10 Rule: 60% neutrals (backgrounds/body text), 30% secondary (cards/headers), 10% accent (buttons/CTAs).



&#x20;   Opacity: Modify color opacity to generate variations rather than introducing new colors.



&#x20;   Contrast Accessibility: Maintain minimum contrast ratios of 3:1 for large text and 4.5:1 for body text. Verify using Figma's native contrast checker.



&#x20;   Extraction: Reverse-engineer existing color palettes using Chrome Developer Tools > CSS Overview.



4\. Code \& AI Implementation



&#x20;   Core Stack: Learn foundational HTML, CSS, JavaScript, and basic PHP (for WordPress).



&#x20;   80/20 Rule: Do not write boilerplate from scratch. Source 80% of the functional code using CodePen, snippets, or AI tools like ChatGPT. Manually tweak the remaining 20% to execute exact project requirements.



5\. Conversion Optimization



&#x20;   Single Objective: Restrict every page to one primary goal (e.g., purchase, consultation, lead capture).



&#x20;   CTA Placement: Position Call-to-Actions in the hero section, the navigation menu, and at intervals corresponding to every 2-3 seconds of scroll depth.



&#x20;   Trust Elements: Validate user intent through clear copywriting, testimonials, and social proof.
